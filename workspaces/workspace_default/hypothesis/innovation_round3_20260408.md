# 三模型协同创新改进计划（Round 3）

## 一、XGBoost 模型改进计划

### 模型定位
**浅层统计特征专家** —— 擅长捕捉人工规则特征（工作日/周末、温度、峰谷标记、节气、周期规律）

### 核心创新点

#### 1. 周期稀疏特征处理（尖端预测 a）
- **特征设计**：构建「周期掩码特征」，不直接在原始特征里加周期标记，而是在 XGBoost 分裂节点时加入周期约束
- **实现方式**：
  - 周末特征：仅取近 4 周周末数据构建独立分裂路径
  - 节假日特征：构建节假日/工作日二元掩码，强制树优先按掩码分裂
  - 节气特征：将 24 节气映射为 12 组阴阳特征对，每组作为独立分裂候选
- **数据适配**：山东负荷数据为小时级，可提取 `hour_of_day`, `day_of_week`, `is_holiday`, `solar_term_id`

#### 2. 困难样本聚焦训练（尖端预测 b）
- **样本加权策略**：使用 `sample_weight` 参数
  - 日内尖峰时段（负荷 > 95 分位数）：权重 = 5.0
  - 高峰时段（负荷 > 75 分位数）：权重 = 3.0
  - 平段（25-75 分位数）：权重 = 0.5
  - 谷段（负荷 < 25 分位数）：权重 = 0.3
- **尖峰检测**：基于历史负荷的 99 分位数斜率阈值识别突变样本

#### 3. 多任务双输出分支（尖端预测 c）
- **架构改造**：搭建双目标 XGBoost
  - 主分支：负荷预测（回归）`objective="reg:squarederror"`
  - 副分支：峰/谷/平分类（分类）`objective="multi:softprob"`，共享树结构
- **联合损失**：`L_total = L_reg + 0.3 * L_cls`
- **输出**：未来 24 小时每个时刻的峰/谷/平标记（基于分时电价或负荷分位数定义）

#### 4. 叶子节点特征提取（整体方案创新）
- **输出特征**：提取最后一层叶子节点索引作为浅层统计特征
- **融合方式**：将叶子节点 one-hot 编码后送入融合层 MLP

## 二、TimesNet 模型改进计划

### 模型定位
**频域周期特征专家** —— 擅长捕捉 24h 日周期、7 天周周期、负荷波动频率

### 核心创新点

#### 1. 频域分解增强（适配周期规律）
- **FFT 预处理**：对输入负荷序列做 FFT 分解，提取主导频率分量
  - 日周期频率：1/24 h⁻¹
  - 周周期频率：1/168 h⁻¹
- **特征输出**：将频域分解后的幅值/相位特征作为中层周期特征送入融合层

#### 2. 困难样本聚焦训练（尖端预测 b）
- **损失加权**：在频域损失中加入尖峰权重
  ```python
  loss = MSE(pred, target) + λ * spike_weight * MSE(FFT(pred), FFT(target))
  ```
- **尖峰增强**：对日内固定尖峰时段（如早高峰 8-10 点、晚高峰 18-22 点）的样本在频域给予更高关注

#### 3. 多任务输出（尖端预测 c）
- **辅助头**：在频域特征后接一个小的分类头，输出峰/谷/平概率
- **周期一致性约束**：强制预测的 24h 曲线满足日周期平滑性

#### 4. 动态滑动窗口（通用改动）
- **自适应窗口**：根据负荷波动率动态调整输入窗口长度
  - 平稳期：窗口 = 72h（3 天）
  - 波动期：窗口 = 168h（7 天）
- **波动率检测**：基于滑动标准差/均值比值判断

### 特征处理流程（结合山东负荷数据集）

| 步骤 | 输入 | 处理方式 | 输出 |
|:---|:---|:---|:---|
| 1. 原始特征 | 负荷 (MW)、时间戳 | 归一化 + 时间编码 | 标准化负荷序列 |
| 2. FFT 分解 | 标准化负荷序列 | 快速傅里叶变换 | 频域幅值谱 + 相位谱 |
| 3. 主导频率提取 | 幅值谱 | 提取前 K 个峰值频率 (K=8) | 主导频率索引 + 幅值 |
| 4. 周期特征构建 | 主导频率 | 逆 FFT 重构 + 原始序列残差 | 周期分量 + 残差分 |
| 5. 融合层输出 | 周期分量 + 残差 | 全局池化 → 128 维向量 | **TimesNet 特征向量** |

### 输出到融合层的接口设计

TimesNet 模块同时输出两类内容到融合层：

**A. 预测输出（用于实时/日前预测）**
- 未来 24 小时负荷预测值（逐小时）
- 频域周期置信度分数（0-1，表示当前预测对周期规律的依赖程度）

**B. 特征输出（用于融合层拼接）**
- 128 维频域特征向量：包含主导频率幅值、相位、周期分量统计量
- 32 维残差特征向量：包含非周期波动的能量分布

```python
# TimesNet 输出接口示例
class TimesNetOutput:
    predictions: np.ndarray      # shape: (24,) 未来 24 小时预测
    frequency_features: np.ndarray  # shape: (128,) 频域特征
    residual_features: np.ndarray   # shape: (32,) 残差特征
    cycle_confidence: float      # 周期置信度 [0, 1]
```

## 三、PatchTST 模型改进计划

### 模型定位
**长时序突变特征专家** —— 擅长捕捉历史长期依赖、尖峰突变、突发波动

### 核心创新点

#### 1. Patch 分块自适应调整
- **自适应 Patch Size**：
  - 平稳段：patch_size = 24（天粒度）
  - 波动段：patch_size = 6（6 小时粒度，捕捉细节）
  - 尖峰段：patch_size = 3（3 小时粒度，精细捕捉突变）
- **切换逻辑**：基于局部方差动态选择 patch 大小

#### 2. 长时序依赖增强（尖端预测 a 的变体）
- **稀疏注意力**：在 Transformer 编码器中使用稀疏注意力机制，只关注：
  - 历史同期（上周同日、上月同日）
  - 历史相似负荷模式片段
- **记忆库**：构建历史尖峰模板库，通过相似度检索辅助预测

#### 3. 困难样本聚焦训练（尖端预测 b）
- **对比学习预训练**（通用改动）：
  - 预训练阶段：构造正负样本对
    - 正样本：同一尖峰模式的不同时间段
    - 负样本：平稳段 vs 尖峰段
  - 微调阶段：加载预训练权重，用加权 MSE 微调
- **尖峰感知位置编码**：对尖峰时段的位置编码加入额外偏移

#### 4. 深层时序特征输出（整体方案创新）
- **特征提取**：从 Transformer 编码器输出提取 `[CLS]` 位置特征或全局池化特征
- **突变检测头**：额外输出一个突变概率分数，辅助融合层判断

### 特征处理流程（结合山东负荷数据集）

| 步骤 | 输入 | 处理方式 | 输出 |
|:---|:---|:---|:---|
| 1. Patch 分块 | 120 小时负荷序列 | 按 patch_size 切分 | 5-40 个 patch（依大小而定） |
| 2. Patch 嵌入 | 每个 patch | 线性投影 → hidden_dim | patch 嵌入向量 |
| 3. 位置编码 | patch 嵌入 | 可学习位置编码 + 尖峰偏移 | 带位置信息的 patch 表示 |
| 4. Transformer 编码 | 带位置编码的 patch | 稀疏自注意力 × 3 层 | 深层时序上下文表示 |
| 5. 特征汇聚 | 编码器输出 | `[CLS]` 向量 + 全局池化 | **PatchTST 深层特征向量** |

### 深层时序特征输出细化

PatchTST 的深层时序特征包含以下层次的信息，通过多级汇聚输出到融合层：

**层次 1：[CLS] 全局语义向量（64 维）**
- 位于 Transformer 输入序列首位置的 `[CLS]` token 经过 3 层自注意力后的输出
- 捕获整个输入序列的全局语义表示
- 用于判断当前负荷处于「平稳期」「波动期」还是「尖峰期」

**层次 2：Patch 级局部特征（96 维）**
- 对最后 4 个 patch 的编码器输出做注意力池化
- 捕获近期负荷的局部变化趋势
- 包含：短期斜率、加速度、波动方向

**层次 3：突变检测分数（标量）**
- 通过一个小型 MLP 头从 `[CLS]` 向量预测突变概率
- 输出范围 [0, 1]，>0.7 判定为尖峰突变
- 用于融合层的门控权重调整

**层次 4：记忆库检索权重（K 维，K=模板数量）**
- 当前输入与历史尖峰模板库的相似度分布
- 用于检索最相似的历史模式辅助预测

```python
# PatchTST 输出接口示例
class PatchTSTOutput:
    predictions: np.ndarray        # shape: (24,) 未来 24 小时预测
    cls_features: np.ndarray       # shape: (64,) 全局语义向量
    local_features: np.ndarray     # shape: (96,) 局部趋势特征
    spike_probability: float       # 突变概率 [0, 1]
    memory_weights: np.ndarray     # shape: (K,) 历史模板相似度
```

### 输出到融合层的接口设计

PatchTST 模块同时输出两类内容到融合层：

**A. 预测输出（用于实时/日前预测）**
- 未来 24 小时负荷预测值（逐小时）
- 突变概率分数（用于融合层门控调整）

**B. 特征输出（用于融合层拼接）**
- 64 维 `[CLS]` 全局语义向量
- 96 维局部趋势特征向量
- K 维历史模板相似度（可选，K=16）

## 四、融合层设计（整体方案创新）

### 特征拼接
```
特征向量 = [
    XGBoost 叶子节点特征 (浅层统计，~64 维),
    TimesNet 频域特征 (中层周期，~128 维),
    PatchTST 时序特征 (深层依赖，~96 维)
]  # 总计 ~288 维
```

### MLP 融合
```python
融合层 = Sequential(
    Linear(288, 128),
    GELU(),
    Dropout(0.1),
    Linear(128, 64),
    GELU(),
    Linear(64, 1)  # 最终负荷预测
)
```

### 多任务输出
```python
辅助分类头 = Sequential(
    Linear(64, 32),
    GELU(),
    Linear(32, 3)  # 峰/谷/平
)
```

## 五、备选方案：误差修正模式

> 此方案可作为备选，先不急于实施

### 核心思路
- 深度学习模型（TimesNet + PatchTST）负责抓复杂时序规律，但尖峰容易预测偏低/偏高
- XGBoost 不直接预测负荷，专门预测「深度学习模型错了多少」
- 最终预测 = 深度学习预测 + XGBoost 误差修正

### 实现步骤
1. 训练 TimesNet + PatchTST 得到基础预测
2. 计算误差 = 真实值 - 基础预测
3. 训练 XGBoost 预测误差（输入：基础预测值 + 原始特征）
4. 推理时：最终输出 = 深度学习输出 + XGBoost 误差修正

## 六、三段话工作纲领

### 纲领一：XGBoost —— 规则驱动的快速响应者
> XGBoost 作为浅层统计特征专家，核心任务是快速捕捉人工可解释的周期规律和外部事件影响。改进重点在于：(1) 构建周期掩码特征，强制树模型在分裂时优先匹配历史同周末、同节假日、同节气的样本；(2) 实施尖峰样本加权训练，给日内尖峰时段样本权重设为 3~5，让树模型优先拟合尖峰，解决被大量平稳样本带偏的问题；(3) 探索双任务输出架构，主分支预测负荷、副分支预测峰谷平分类，共享树结构提升尖峰判别能力。最终输出叶子节点特征作为浅层统计信号送入融合层。

### 纲领二：TimesNet —— 频域周期的稳定捕捉者
> TimesNet 作为频域周期特征专家，核心任务是精准建模 24 小时日周期和 7 天周周期的稳定规律。改进重点在于：(1) 引入 FFT 频域分解，提取主导频率分量作为中层周期特征；(2) 在频域损失中加入尖峰权重，对固定尖峰时段（早晚高峰）给予更高关注；(3) 实施动态滑动窗口策略，根据负荷波动率自适应调整输入窗口长度（平稳期 72h、波动期 168h）。最终输出频域特征作为周期稳定性信号送入融合层。

### 纲领三：PatchTST —— 长时依赖的敏锐感知者
> PatchTST 作为长时序突变特征专家，核心任务是捕捉历史长期依赖关系和突发尖峰波动。改进重点在于：(1) 实现 Patch 分块自适应调整，根据局部方差动态选择 patch 大小（平稳段 24h、波动段 6h、尖峰段 3h）；(2) 引入稀疏注意力机制，只关注历史同期和相似负荷模式片段，降低计算复杂度；(3) 采用对比学习预训练 + 微调范式，预训练阶段学习尖峰模式表示，微调阶段用加权 MSE 聚焦尖峰拟合。最终输出 Transformer 编码器特征作为深层时序信号送入融合层。

## 八、各模型特征处理流程（结合山东负荷数据集）

### XGBoost 特征处理

| 步骤 | 输入特征 | 处理方式 | 输出维度 |
|:---|:---|:---|:---|
| 1. 时间编码 | 时间戳 | `hour`, `day_of_week`, `month`, `is_weekend`, `is_holiday` | 5 维 |
| 2. 节气编码 | 日期 | 24 节气映射为 12 组阴阳特征对 | 12 维 |
| 3. 周期掩码 | 日期 + 节假日 | 周末掩码、节假日掩码、工作日掩码 | 3 维 |
| 4. 负荷统计 | 历史负荷 | `lag_24`, `lag_168`, `rolling_mean_24`, `rolling_std_24` | 4 维 |
| 5. 天气融合 | 温度、湿度 | 温度分段编码（<0°C, 0-25°C, >25°C）+ 湿度分位 | 4 维 |
| 6. 峰谷标记 | 历史负荷分位 | 基于分时电价或负荷百分位数的峰/谷/平标记 | 2 维（one-hot） |
| **总计输入** | | | **~30 维** |
| **叶子节点输出** | 树模型最后一层 | 50 棵树 × 每树 4 叶子 → one-hot 编码 | **~200 维 → 压缩至 64 维** |

### TimesNet 特征处理

| 步骤 | 输入特征 | 处理方式 | 输出维度 |
|:---|:---|:---|:---|
| 1. 负荷归一化 | 原始负荷 (MW) | Z-score 标准化 | 1 维序列 |
| 2. 时间嵌入 | 时间戳 | sin-cos 周期编码（24h、168h 周期） | 4 维 |
| 3. FFT 分解 | 归一化负荷序列 | 快速傅里叶变换 → 幅值谱 + 相位谱 | 频域表示 |
| 4. 主导频率提取 | 幅值谱 | Top-K 峰值频率 (K=8) + 逆 FFT 重构 | 周期分量 + 残差 |
| 5. 特征汇聚 | 周期分量 + 残差 | 全局池化 + 全连接投影 | **128 维 + 32 维** |

### PatchTST 特征处理

| 步骤 | 输入特征 | 处理方式 | 输出维度 |
|:---|:---|:---|:---|
| 1. Patch 分块 | 120 小时负荷序列 | 自适应 patch_size (3/6/24) | 5-40 个 patch |
| 2. Patch 嵌入 | 每个 patch | 线性投影 → hidden_dim (96) | patch 嵌入向量 |
| 3. 位置编码 | patch 嵌入 | 可学习位置编码 + 尖峰偏移 | 带位置的 patch 表示 |
| 4. 稀疏注意力 | patch 序列 | 3 层 Transformer 编码器 | 深层上下文表示 |
| 5. 特征汇聚 | 编码器输出 | `[CLS]` 向量 (64 维) + 局部池化 (96 维) | **160 维** |

## 九、融合层处理逻辑详解

### 融合层输入输出流程

```
┌─────────────────────────────────────────────────────────────────┐
│                     三模型并行推理                              │
├─────────────────┬─────────────────────┬─────────────────────────┤
│   XGBoost       │    TimesNet         │    PatchTST             │
│                 │                     │                         │
│ 输入：~30 维特征 │  输入：120h 负荷序列   │  输入：120h 负荷序列      │
│                 │                     │                         │
│ 输出 A: 负荷预测 │  输出 A: 负荷预测     │  输出 A: 负荷预测        │
│ 输出 B: 64 维特征 │  输出 B: 160 维特征    │  输出 B: 160 维特征       │
└────────┬────────┴──────────┬──────────┴──────────┬──────────────┘
         │                   │                     │
         ▼                   ▼                     ▼
┌─────────────────────────────────────────────────────────────────┐
│                      融合层处理                                 │
│                                                                 │
│  1. 特征拼接：[64 维 XGB + 128 维 TimesNet + 96 维 PatchTST] = 288 维  │
│                                                                 │
│  2. 门控加权（可选）：                                          │
│     - 使用 TimesNet 的 cycle_confidence 加权频域特征             │
│     - 使用 PatchTST 的 spike_probability 调整时序特征权重         │

│                                                                 │
│  3. MLP 融合：288 → 128 → 64 → 1                                │
│                                                                 │
│  4. 多任务输出：                                                 │
│     - 主任务：最终负荷预测（回归）                               │
│     - 辅助任务：峰/谷/平分类（分类）                            │
└─────────────────────────────────────────────────────────────────┘
         │
         ▼
┌─────────────────────────────────────────────────────────────────┐
│                     最终输出                                    │
│  - 未来 24 小时负荷预测值（逐小时）                              │
│  - 未来 24 小时峰/谷/平标记                                    │
│  - 融合置信度分数（用于不确定性估计）                           │
└─────────────────────────────────────────────────────────────────┘
```

### 融合层代码框架

```python
import torch
import torch.nn as nn

class EnsembleFusionLayer(nn.Module):
    def __init__(self, xgb_dim=64, timesnet_dim=160, patchtst_dim=160, 
                 hidden_dim=128, num_classes=3):
        super().__init__()
        
        # 特征拼接维度
        concat_dim = xgb_dim + timesnet_dim + patchtst_dim
        
        # 门控网络（可选）
        self.gate_net = nn.Sequential(
            nn.Linear(3, 3),  # 输入：3 个模型的置信度
            nn.Softmax(dim=-1)
        )
        
        # MLP 融合
        self.fusion_mlp = nn.Sequential(
            nn.Linear(concat_dim, hidden_dim),
            nn.GELU(),
            nn.Dropout(0.1),
            nn.Linear(hidden_dim, hidden_dim // 2),
            nn.GELU(),
            nn.Dropout(0.1),
        )
        
        # 主任务头：负荷预测
        self.regression_head = nn.Linear(hidden_dim // 2, 1)
        
        # 辅助任务头：峰谷平分类
        self.classification_head = nn.Sequential(
            nn.Linear(hidden_dim // 2, 32),
            nn.GELU(),
            nn.Linear(32, num_classes)
        )
    
    def forward(self, xgb_feat, timesnet_feat, patchtst_feat,
                xgb_pred, timesnet_pred, patchtst_pred,
                confidences=None):
        """
        Args:
            xgb_feat: [B, 64] XGBoost 叶子节点特征
            timesnet_feat: [B, 160] TimesNet 频域特征
            patchtst_feat: [B, 160] PatchTST 时序特征
            xgb_pred, timesnet_pred, patchtst_pred: [B, 24] 各模型预测
            confidences: [B, 3] 各模型置信度（可选）
        
        Returns:
            final_pred: [B, 24] 最终负荷预测
            peak_valley_pred: [B, 24, 3] 峰谷平分类概率
            fused_features: [B, 64] 融合后特征
        """
        # 1. 特征拼接
        fused = torch.cat([xgb_feat, timesnet_feat, patchtst_feat], dim=-1)
        
        # 2. 门控加权（可选）
        if confidences is not None:
            gate_weights = self.gate_net(confidences)
            # 按置信度加权各模型特征
            fused = fused * gate_weights.sum()
        
        # 3. MLP 融合
        fused_features = self.fusion_mlp(fused)
        
        # 4. 多任务输出
        final_pred = self.regression_head(fused_features)
        peak_valley_pred = self.classification_head(fused_features)
        
        return final_pred, peak_valley_pred, fused_features
```

### 评估指标体系

| 指标类别 | 指标名称 | 公式/说明 | 目标值 |
|:---|:---|:---|:---|
| **点预测精度** | MAE | 平均绝对误差 | < 3.0 |
| | RMSE | 均方根误差 | < 4.0 |
| | SMAPE | 对称平均绝对百分比误差 | < 5% |
| | WAPE | 加权平均百分比误差 | < 5% |
| **尖峰预测** | Peak-MAE | 尖峰时段（Top 10%）的 MAE | < 5.0 |
| | Spike-Capture-Rate | 尖峰检出率（预测命中/实际尖峰数） | > 80% |
| **分类任务** | Peak-Valley-Accuracy | 峰/谷/平分类准确率 | > 75% |
| | Macro-F1 | 三类别宏平均 F1 分数 | > 0.70 |
| **融合增益** | Fusion-Gain-MAE | (最佳单模型 MAE - 融合 MAE) / 最佳单模型 MAE | > 10% |
| | Diversity-Score | 三模型预测结果的方差/均值 | 0.1-0.3 |
