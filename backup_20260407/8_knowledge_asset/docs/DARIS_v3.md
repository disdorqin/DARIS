## 【第二部分：OpenClaw 全局调度 + 钉钉全链路管控优化（强制落地）】

### 核心优化目标

将 OpenClaw 作为项目唯一全局调度入口，主文件放置在项目根目录，实现「启动即连通网管 + 接通钉钉 + 进入指令监听状态」，支持通过钉钉对话全流程操控项目，所有执行状态、报错告警实时返回钉钉。

### 强制执行步骤（严格按顺序落地）

#### 1. OpenClaw 主文件体系重构（根目录部署）

在项目根目录创建 OpenClaw 核心主文件，结构如下：

```
DARIS/
├── openclaw_main.py              # 【唯一启动入口】OpenClaw 全局调度主文件（根目录）
├── openclaw_requirements.txt     # OpenClaw 专属依赖清单
├── 1_config/base/openclaw_config.yaml    # OpenClaw 全局配置文件
└── 1_config/alert/dingtalk_config.yaml   # 钉钉机器人专属配置
```

#### 2. openclaw_main.py 核心功能强制实现

必须在主文件中实现以下核心模块，每个模块必须输出开发完成日志与可用性校验结果：

**网管连接模块**：自动读取.env 内的大模型 API 配置、OpenClaw 环境配置，启动时自动完成网管连接、OpenClaw 服务初始化、环境适配，输出连接成功/失败日志，失败自动重试 3 次

**钉钉机器人对接模块**：自动读取 dingtalk_config.yaml 内的 Webhook 地址、加签密钥，实现双向能力：
- 告警推送：全流程环节状态、分级告警、执行日志实时推送至钉钉
- 指令监听：持续监听钉钉群内的指令消息，识别后触发对应操作

**钉钉指令集强制实现**，必须支持以下指令，识别后立即执行：

| 钉钉指令 | 触发动作 | 执行后钉钉返回内容 |
|----------|----------|----------------------|
| `/start` | 启动一轮完整的 DARIS 科研闭环工作流 | 工作流启动通知、当前环节实时进度推送 |
| `/pause` | 暂停当前执行的工作流 | 暂停成功通知、当前执行进度快照 |
| `/resume` | 恢复暂停的工作流 | 恢复成功通知、继续执行进度推送 |
| `/rollback` | 回滚到上一稳定版本 | 回滚成功通知、回滚版本详情 |
| `/status` | 查询当前项目执行状态 | 当前环节、进度、资源使用情况、最新实验指标 |
| `/stop` | 终止当前执行的工作流 | 终止成功通知、本轮已完成内容总结 |
| `/clean` | 触发项目冗余文件自动清理 | 清理清单、清理结果、当前项目空间占用 |

**全流程工作流调度模块**：对接项目 8 个智能体环节，实现环节自动流转、异常自动处理、状态实时同步，所有环节的触发、执行、校验均由 OpenClaw 统一管控

**告警分级推送模块**：严格按照以下规则推送钉钉消息，必须 100% 落地：
- 紧急告警：服务器失联、训练中断、代码报错 3 次修复失败、克隆仓库失败，立即推送
- 预警告警：资源接近阈值、指标连续 3 轮下降、环节执行超时，10 分钟内推送
- 通知告警：环节完成、迭代结束、最优模型生成、指令执行完成，阶段推送

**持久化记忆模块**：自动同步项目 `8_knowledge_asset/iteration_memory/` 记忆库，每轮工作流完成后自动更新 skills 知识库、深度复盘内容，实现能力迭代
