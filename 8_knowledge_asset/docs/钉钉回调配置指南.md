# 钉钉回调配置指南

## 概述

本文档说明如何配置钉钉回调功能，实现通过钉钉机器人双向控制 DARIS 系统。

## 配置步骤

### 步骤 1：创建钉钉企业内部应用

1. 登录 [钉钉开发者后台](https://open-dev.dingtalk.com/)
2. 点击「应用开发」→「企业内部应用」→「创建应用」
3. 填写应用信息：
   - 应用名称：DARIS 控制器
   - 应用图标：可选
   - 应用描述：DARIS 系统钉钉控制器

### 步骤 2：获取应用凭证

创建应用后，在「凭证管理」页面获取以下信息：
- **AppKey**：应用的唯一标识
- **AppSecret**：应用的密钥

将这些值填入 `.env` 文件：
```bash
DINGTALK_APP_KEY=你的 AppKey
DINGTALK_APP_SECRET=你的 AppSecret
```

### 步骤 3：配置回调 URL

1. 在应用管理页面，点击「开发管理」→「回调配置」
2. 配置回调地址：
   ```
   URL: http://47.100.98.160:8080/
   Token: 生成一个随机字符串（如：daris2024token）
   ```
3. 将 Token 填入 `.env` 文件：
   ```bash
   DINGTALK_TOKEN=daris2024token
   ```

### 步骤 4：配置事件订阅

1. 在「开发管理」→「事件订阅」中启用以下事件：
   - 接收消息
   - 机器人消息

2. 选择「加密方式」：
   - 如果启用加密，需要配置 AES 密钥
   - 建议初期使用不加密模式

### 步骤 5：发布应用

1. 在「版本管理」中发布应用
2. 添加可用范围（选择包含你的钉钉群）

### 步骤 6：添加到钉钉群

1. 在钉钉群中点击「机器人」→「添加机器人」
2. 选择「企业内部应用」→ 选择「DARIS 控制器」
3. 完成添加

## 服务器配置

### 防火墙设置

确保服务器 8080 端口开放：

```bash
# 阿里云服务器安全组配置
# 添加入站规则：端口 8080，协议 TCP，来源 0.0.0.0/0
```

### 启动服务

```bash
# 启动 OpenClaw（带钉钉回调）
python openclaw_main.py --daemon --callback-port 8080

# 仅启动回调服务器（不执行工作流）
python 2_agent_system/dingtalk_callback.py --port 8080
```

## 测试

### 测试回调连接

1. 在钉钉群发送：`/help`
2. 应该收到响应：可用指令列表

### 测试指令

| 指令 | 功能 | 预期响应 |
|------|------|----------|
| `/start` | 启动工作流 | 🚀 DARIS 工作流已启动 |
| `/status` | 查询状态 | 📊 DARIS 当前状态 |
| `/pause` | 暂停工作流 | ⏸️ 工作流已暂停 |
| `/resume` | 恢复工作流 | ▶️ 工作流已恢复 |
| `/stop` | 停止工作流 | ⏹️ 工作流已终止 |
| `/clean` | 清理文件 | 🧹 清理完成 |
| `/rollback` | 版本回滚 | 🔄 已回滚到版本 |
| `/help` | 显示帮助 | 📖 DARIS 指令帮助 |

## 故障排查

### 回调不工作

1. 检查服务器端口是否开放：
   ```bash
   netstat -an | grep 8080
   ```

2. 检查防火墙设置：
   ```bash
   # Windows
   netsh advfirewall firewall add rule name="DingTalk" dir=in action=allow protocol=TCP localport=8080
   ```

3. 查看回调日志：
   ```
   7_monitor_system/dingtalk_log/
   ```

### 签名验证失败

确保 `.env` 中的 `DINGTALK_TOKEN` 与钉钉后台配置的一致。

### 消息无法发送

检查 Webhook URL 是否正确：
```bash
# 测试 Webhook
curl -X POST "https://oapi.dingtalk.com/robot/send?access_token=YOUR_TOKEN" \
  -H "Content-Type: application/json" \
  -d '{"msgtype":"text","text":{"content":"test"}}'
```

## 安全建议

1. **Token 安全**：使用强随机字符串作为 Token
2. **IP 白名单**：在钉钉后台配置回调 IP 白名单
3. **HTTPS**：生产环境建议使用 HTTPS（需要配置 SSL 证书）
4. **权限控制**：限制可执行指令的用户

## 高级配置

### 使用 HTTPS

1. 获取 SSL 证书（可使用 Let's Encrypt 免费证书）
2. 修改回调服务器配置使用 HTTPS
3. 更新钉钉后台的回调 URL 为 HTTPS 地址

### 消息加密

如果启用消息加密：
1. 生成 AES 密钥（32 位随机字符串）
2. 在钉钉后台配置加密密钥
3. 在 `.env` 中配置：
   ```bash
   DINGTALK_ENCODING_AES_KEY=你的 32 位密钥
   ```
